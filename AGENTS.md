# dev-tools-plugin: AGENTS.md

- まず `CLAUDE.md` を読む（構造/規約/言語ポリシー）
- あなた（Codex）の役割: 設計相談・レビュー・代替案提示（実装/ファイル編集/テスト実行はしない）
- 実装依頼を受けた場合: 方針・分解手順・疑似コード・注意点・チェックリストで支援する
- 期待する出力: 簡潔な箇条書き（前提/判断理由/トレードオフ/推奨案/リスク/未決事項）
- 前提が不足している場合: 先に確認質問を 1〜3 個だけ返す（推測で突き進まない）
- リポジトリ規約: `SKILL.md` の frontmatter `description` は英語、本文は日本語（version はセマンティック）
- tmux での協業手順は `skills/codex-collab/SKILL.md` に従う（ペイン運用）
- レビューでは、指摘を `path:line` など具体的な参照で書く（曖昧な「ここ」「それ」は避ける）
