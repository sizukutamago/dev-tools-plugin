# shell-debug

シェルスクリプトやワンライナーの不具合を体系的にデバッグするためのスキル。

## 概要

このスキルは、シェルの不具合を「再現→分解→観測→修正→検証」のワークフローで最短特定するための手順集です。

**対象:**
- パイプライン（awk/sed/grep 混在）
- クォート/変数展開の問題
- tmux 自動化スクリプト
- git/CI スクリプト

## 期待するアウトプット

1. 再現コマンド（最小入力つき）
2. 期待出力 / 実際出力（差分が分かる形）
3. 失敗点の切り分け結果（どの段/どの展開で壊れるか）
4. 修正案（最小変更）
5. 検証手順（quick / full）
6. 再発防止（必要なら kaizen へ引き渡し）

## kaizen への引き渡し基準

以下の場合は `/kaizen` で再発防止を検討:

- 同種事故が2回以上発生
- 手動検証が残っている（自動化の余地あり）
- ワンライナーが肥大化（関数化/スクリプト化の余地あり）

**引き渡し時の情報**:
- 発生した問題の概要
- 根本原因
- 提案する標準化/テンプレ化の方向性

## 関連スキル

| スキル | 用途 | 使い分け |
|--------|------|----------|
| `/shell-debug` | 「切り分け実務」に全振り | 壊れている/期待と違うを最短特定 |
| `/kaizen` | 「再発防止・標準化」 | shell-debug の最後から引き継ぎ |
| `/tdd-integration` | テスト駆動開発 | golden-test 化する場合に連携 |
